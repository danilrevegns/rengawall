# Rengawall: пол и отделочные стены по контуру помещения в Renga

Программа на Python подключается к открытому проекту **Renga** и для выбранных **помещений** создаёт **пол** и **стены** вдоль контура, рассчитанного по геометрии помещения. Толщины, высота стен и включение пола/стен задаются в файле настроек и могут зависеть от **трёх пользовательских свойств** помещения (например, тип комнаты по классификатору проекта).

Запуск выполняется **через графическое окно** (tkinter): путь к настройкам и режим обработки выбираются в интерфейсе.

Официальная документация API Renga: [https://help.rengabim.com/api/](https://help.rengabim.com/api/)

---

## Что нужно заранее

| Требование | Зачем |
|------------|--------|
| **Windows** | Связь с Renga идёт через COM (как в документации Renga). |
| **Установленная Renga** | Программа подключается к **уже запущенной** Renga с открытым проектом. |
| **Python 3.8 или новее** | См. раздел «Установка Python». |
| **Интернет** | Один раз — для установки `comtypes` через `pip`. |

Библиотека **tkinter** для окна обычно уже входит в стандартную установку Python для Windows.

## Установка Python (если его ещё нет)

1. Откройте сайт [https://www.python.org/downloads/](https://www.python.org/downloads/) и скачайте установщик для Windows.
2. Запустите установщик. **Обязательно** отметьте галочку **«Add python.exe to PATH»** («Добавить Python в PATH») и нажмите «Install Now».
3. Проверка: откройте **PowerShell** или **«Командная строка»** (Win + R → `cmd` или `powershell` → Enter) и выполните:

   ```text
   python --version
   ```

   Должно появиться что-то вроде `Python 3.12.x`. Если команда не найдена — Python не добавлен в PATH.

Если команда `python` не работает, попробуйте `py --version` и дальше вместо `python` используйте `py`.

## Шаг 1. Папка проекта

Все файлы должны лежать в одной папке, например:

`C:\Users\ВашеИмя\PycharmProjects\rengawall`

Нужны как минимум:

- `rengawalll.py` — программа;
- `requirements.txt` — зависимости;
- `room_finish_config.example.json` — пример настроек (из него делают рабочий JSON).

---

## Шаг 2. Установить зависимости (один раз)

1. Откройте **PowerShell** или **Командную строку**.
2. Перейдите в папку проекта (путь замените на свой):

   ```text
   cd C:\Users\ВашеИмя\PycharmProjects\rengawall
   ```

3. Выполните:

   ```text
   python -m pip install -r requirements.txt
   ```

   При необходимости:

   ```text
   py -3 -m pip install -r requirements.txt
   ```

Устанавливается **comtypes** — без неё связь с Renga не работает. При первом запуске с открытой Renga генерируется модуль `comtypes.gen.Renga` (типы `Point2D`, интерфейсы).

## Шаг 3. Файл настроек JSON

### 3.1. Создать файл из примера

1. Скопируйте `room_finish_config.example.json`.
2. Переименуйте копию в **`room_finish_config.json`** (или другое имя — тогда укажете его в окне программы).

### 3.2. Поле `property_ids`

В массиве **`property_ids`** — **ровно три** строки-GUID **пользовательских свойств** типа «Помещение» в вашем проекте Renga (не заглушки из примера). Формат: `{xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx}`. Порядок: первый GUID → ключ **`"0"`** в `match`, второй → **`"1"`**, третий → **`"2"`**.

### 3.3. `defaults` и `rules`

- **`defaults`** — настройки по умолчанию, если ни одно правило не подошло.
- **`rules`** — список правил; в **`match`** задаются ожидаемые значения трёх свойств (сравнение без учёта регистра).

Строка **`"comment"`** программой не читается.

### 3.4. Синтаксис JSON

Проверьте запятые и кавычки; при необходимости используйте проверку JSON в редакторе или валидатор в браузере.

## Шаг 4. Подготовить Renga

1. Запустите **Renga** и откройте **нужный проект**.
2. В модели должны быть **помещения**; у обрабатываемых — по возможности заполнены три свойства из `property_ids`.
3. **Не закрывайте** Renga до окончания работы программы — подключение идёт к уже запущенной сессии.

---

## Шаг 5. Запуск (графическое окно)

1. Откройте **PowerShell** или **Командную строку**.
2. Перейдите в папку проекта:

   ```text
   cd C:\Users\ВашеИмя\PycharmProjects\rengawall
   ```

3. Запустите программу **с флагом графического режима**:

   ```text
   python rengawall.py --gui
   ```

4. Откроется **окно**:

   - В поле пути укажите файл настроек (по умолчанию подставляется `room_finish_config.json`) или нажмите **«…»** и выберите JSON в проводнике.
   - **«Все помещения»** — обработать все помещения в модели.
   - **«Только выбранные в Renga»** — обработать только помещения, **выделенные в активном виде** в Renga (сначала выделите их в Renga, затем нажмите кнопку).

5. Ход работы и сообщения об ошибках отображаются в **текстовом поле** внизу окна. После успешного выполнения при необходимости **сохраните проект в Renga** вручную.

## Если что-то пошло не так

| Симптом | Что проверить |
|--------|----------------|
| Окно не открылось, ошибка про `tkinter` | Переустановите Python с полной стандартной библиотекой или включите компонент tcl/tk в установщике. |
| Нет модуля `comtypes` / ошибка импорта `comtypes.gen.Renga` | `pip install -r requirements.txt`, затем запустите Renga и снова скрипт (нужна генерация typelib). |
| Нет открытого проекта в Renga | Откройте проект в Renga и повторите запуск. |
| «Нет помещений для обработки» | Для режима выбранных объектов ничего не выделено в Renga; в модели нет помещений. |
| Ошибка чтения JSON | Путь к файлу, синтаксис JSON. |
| Ошибки COM / типы | Обновите `comtypes`, Renga должна быть установлена; интерфейсы из `GetInterfaceByName` оборачиваются в `Dispatch`, как в примерах `test_*.py`. |
| Пол без контура | В части версий Renga контур пола через API может задаваться ограниченно — донастройте вручную в Renga. |
